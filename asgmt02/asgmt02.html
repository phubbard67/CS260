Instructions

In this assignment, you will implement a “quack,” which is the result of cross-breeding a queue and a stack.

Your implementation of the Quack class will store its items as a circular array (you can find a description of this concept in the textbook). Your array should be dynamically allocated. The number of array elements should be as specified by the capacity argument of the Quack constructor.

The point of a circular array is that very few array elements need to be moved from their first location in the array to other locations. Each of the functions in your code is required to move the minimum possible number of array elements.

Your code will need to allow the circular array to become “full” (i.e. every slot contains a valid item). Your Quack instance should keep track of when the array is full by remembering the size of the array and the current number of items it contains.

Your implementation of the rotate and reverse member functions should operate directly on the circular array within the function itself, and not make any calls to popFront, pushBack, etc. Your rotate function should take advantage of the properties of a circular array to move as few elements of the array as possible.

You are required to mplement rotate and reverse to work in place in the array pointed to by the items data member. “In place” means in the original array, without using any other array. The only array variable you are allowed to use in these functions is the items data member itself.

You should store the first item you add to the quack in array location 0. Items pushed onto the back should be stored in increasing locations (“increasing” means in the direction of higher indices). Items pushed onto the front should be stored in decreasing locations. This means that if the quack is initially empty and the first two items are added via pushFront, the first one will go into items[0], and the second one will go into items[sizeOfArray – 1].

The driver produces a file named array.txt, which shows where each item is stored in the assignment. Your code will need to produce an array.txt whose contents match one of the provided files depending on which version of the assignment you choose to do.

There are additional instructions in comments in the code files.

This assignment comes in two versions: Regular (which is easier) and Premium (which is more difficult). If you want to get an A in this course, you will need to do the Premium version. If, instead, you will be satisfied with a B, you can do the Regular version and save yourself some work.

Initialization Lists

The constructors in your code should use an initialization list item for each data member defined by the class. Points will be deducted where this is not done.

Here is a class definition:

class SomeClass
{
    ...

private:
    int     dataMember1;
    int     dataMember2;
}

Here is a constructor for this class that does not use an initialization list:

SomeClass::SomeClass(int dataMember1, int dataMember2)
{
    this->dataMember1 = dataMember1;
    this->dataMember2 = dataMember2;
}

And here is one that does use an initialization list: 

SomeClass::SomeClass(int dataMember1, int dataMember2) :
    dataMember1{dataMember1},
    dataMember2{dataMember2}
{
}

An initialization list consists of comma-separated items between the : and the { that begins the body of the function. Each of these items is of the form:

dataMemberName{value}

Each item sets the named data member to the value in the parentheses. In the SomeClass constructor above, each of the two initialization list items sets its data member to the constructor argument named inside the parentheses. This is weird-looking syntax, but it is standard usage in the culture of C++.
